Learning Git and GitHub

Introduction

Git is a distributed version control system that allows developers to track changes in their codebase. GitHub is a platform built on top of Git that provides a collaborative environment for developers to share and manage code. Together, they form a powerful toolset for modern software development.

Whether you are a beginner or an experienced developer, understanding Git and GitHub is essential for collaborating on projects and maintaining a clean code history.

Why Use Git & GitHub?

  • Version Control: Keep track of every change made to your code, allowing you to revert to previous versions if needed.
  • Collaboration: Work with other developers, review code changes, and manage contributions effectively.
  • Backup: Your code is safely stored and can be accessed from anywhere, ensuring you never lose your work.
  • Integration: Integrate with various CI/CD tools for automated testing and deployment, streamlining your development workflow.
  • Community: Join a large community of developers, share your projects, and contribute to open-source software.

Basic Git Commands

Command Description
git init Initialize a new Git repository
git clone [url] Clone an existing repository
git add [file] Stage changes for commit
git commit -m "message" Commit changes with a message
git push Push changes to a remote repository
git pull Fetch and merge changes from a remote repository

Advanced Git Commands

Command Description
git branch List, create, or delete branches
git checkout [branch] Switch to a specified branch
git merge [branch] Merge a specified branch into the current branch
git rebase [branch] Reapply commits on top of another base tip
git log Show the commit history
git reset [commit] Reset the current branch to a specified commit
git stash Stash changes in a dirty working directory

Best Practices

  • Commit Often: Make small, frequent commits to keep track of your progress and make it easier to identify issues.
  • Write Meaningful Commit Messages: Clearly describe what changes were made and why.
  • Use Branches: Create separate branches for new features, bug fixes, and experiments to keep your main branch stable.
  • Review Code: Regularly review code changes with your team to maintain code quality and share knowledge.
  • Stay Updated: Regularly pull updates from the remote repository to stay in sync with your team.
